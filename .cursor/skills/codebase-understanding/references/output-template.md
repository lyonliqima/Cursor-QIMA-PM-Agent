## Notes for the Agent

- Never fabricate repo names, team names, or service paths. If a section has no source-backed content, write "Not found in scan".
- Backtick all repo names and service identifiers because users may copy them into terminal or search.
- Do not paste this report into a PRD body. The PRD links to it from the Section 1 related-materials row.
- Create a second PRD-ready field guide when invoked by `write-prd`; it should summarize Sections 8-9 in product language.
- Do not let the field dictionary become a raw code dump. Every field should answer: "what does the user see or decide because of this?"
