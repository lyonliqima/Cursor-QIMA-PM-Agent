§ IIThree Layers
iOrchestration
SKILL.md — 8 phases · 2 checkpoints · 2 auto-loops. Swap a rule without touching the main flow.
iiReferences
6 rulebooks. SKILL pulls them in at specific phases; the PM never reads them directly.

§ IVDesign Principles
i.
Never fabricate
Every claim must trace back to a source or be explicitly marked “assumption — needs user confirmation”. Short and true beats long and invented.
ii.
Orchestration vs rules
SKILL only decides “when to call what”; rule details live in references. Swap a rule without touching the main flow.
iii.
No silent in-fill
Depth-gate ceiling → Open Questions; missing pattern → AskUserQuestion. The skill never makes product judgment calls for the PM.
